# 需求分析 SOP（目录化重构版 v2.0）

## 文档定位

本 SOP 只覆盖“需求分析阶段”，目标是输出可测试、可交接、可冻结的需求契约。

与旧版相比，v2.0 新增：
- 事实核验机制：先判定批评是否属实，再决定是否采纳。
- 上下文裁剪机制：避免把原始长日志直接喂给审查 Agent。
- 证据路由机制：统一输入输出模板，减少信息漂移。

## 文档结构

1. 01-阶段与门禁.md：阶段流程、输入输出、退出门禁。
2. 02-证据与上下文工程.md：日志裁剪、diff 限幅、提示词封装。
3. 03-角色协作与裁决.md：Owner/执行/审查的责任边界。
4. 04-模板与检查清单.md：可复制模板与 DoD 检查项。

## 执行顺序

1. 完成阶段 0 分流。
2. 按阶段一到阶段三推进，不可跳步。
3. 阶段门禁不满足时，必须回退补齐。
4. 输出《需求契约包》后，方可进入开发协作 SOP。
