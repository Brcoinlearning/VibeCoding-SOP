# 04 I/O 托管与产物路由

## 4.1 基本原则

1. Agent 不负责目录拼写与路径决策。
2. Agent 只输出内容本体和 frontmatter 元数据。
3. 脚本根据元数据决定落盘目录和文件名。

## 4.2 元数据示例

```yaml
---
type: review_report
task_id: T-102
stage: review
status: draft
created_at: 2026-04-10T11:30:00+08:00
---
```

## 4.3 路由规则

- type=requirement_contract -> 20-planning/
- type=execution_evidence -> 30-build/
- type=review_report -> 40-review/
- type=go_no_go_record -> 50-release/

## 4.4 决定性命名

命名规则：{task_id}-{type}-{timestamp}.md

示例：T-102-review_report-20260410-1130.md

## 4.5 校验机制

落盘前必须执行：
1. frontmatter 字段完整性检查。
2. 路由映射有效性检查。
3. 同名冲突处理（追加时间戳）。

校验失败时，脚本拒绝写入并返回错误。
