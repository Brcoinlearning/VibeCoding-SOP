# 01 执行架构与角色

## 1.1 架构概览

执行链路：
1. Owner 下发 task 契约。
2. Builder 执行并产出变更与验证证据。
3. 编排脚本自动收集证据并注入 Reviewer。
4. Reviewer 输出结构化审查结论。
5. Owner 做最终 Go/No-Go 裁决。

## 1.2 角色边界

Owner：
- 负责范围冻结、风险接受、最终放行。

Builder：
- 负责实现、测试、提交证据。
- 不得自评通过。

Reviewer：
- 负责独立审查与风险分级。
- 不得直接改需求目标。

Orchestrator（脚本层）：
- 负责事件监听、数据裁剪、上下文封装、I/O 路由。
- 不参与业务决策。

## 1.3 RACI

| 活动 | Owner | Builder | Reviewer | Orchestrator |
|---|---|---|---|---|
| task 契约确认 | A | R | C | I |
| 开发与测试 | I | R | C | I |
| 证据打包与注入 | I | C | I | R |
| 独立审查 | I | I | R | C |
| Go/No-Go | A | I | C | I |
