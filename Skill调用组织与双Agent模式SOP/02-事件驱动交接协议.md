# 02 事件驱动交接协议

## 2.1 为什么要改

禁止手工跨窗口复制以下内容：
- 长日志
- 大块 diff
- 审查报告

原因：手工复制会引入遗漏、版本错位和时间延迟。

## 2.2 触发事件

1. build.completed：Builder 完成本轮提交并产生 diff。
2. test.completed：测试完成并产出通过/失败摘要。
3. review.completed：Reviewer 输出结构化结论。

## 2.3 自动交接流程

1. 监听器捕获 build.completed + test.completed。
2. 裁剪器执行日志摘要和 diff 限幅。
3. 封装器按统一模板生成 reviewer_input。
4. 注入器将 reviewer_input 发送到隔离上下文中的 Reviewer。
5. 结果写入审查产物并通知 Owner。

## 2.4 人类介入点

仅保留两个人工介入点：
- Interrupt：脚本检测到异常（证据缺失、格式错误、执行失败）。
- Approve：Owner 最终放行。

## 2.5 失败回退

以下任一触发即回退：
- 证据不新鲜。
- 关键输入缺失。
- 审查输出非结构化或字段缺失。

回退动作：
1. 标记当前轮次失败。
2. Builder 重新补证据。
3. 重新触发审查。
