# 05 操作模板与运行清单

## 5.1 Owner -> Builder 模板

- task_id：
- 目标：
- In Scope：
- Out of Scope：

- 风险等级：
- 要求：按 TDD 执行并提交新鲜证据。

## 5.2 自动注入给 Reviewer 的输入模板

```xml
<task_contract>
...任务契约摘要...
</task_contract>
<diff_summary>

</diff_summary>
```

## 5.3 Reviewer -> Owner 输出模板

- 结论：Go / No-Go / Go-with-risk
- 分级：Blocker / Major / Minor
- 致命问题：
- 复现路径：
- 影响范围：
- 证据引用：
- 建议动作：

## 5.4 Owner 裁决模板

- 决策：Go / No-Go
- 原因：
- 风险接受：是/否
- 后续动作：
- 截止时间：

## 5.5 一页运行清单

1. 下发 task 契约。
2. Builder 执行并提交证据。
3. 脚本自动裁剪并注入 Reviewer。
4. Reviewer 输出结构化结论。
5. Owner 做最终裁决并归档。
